Speaker: David Grove (IBM Research)

Two talks:
&nbsp; - A tutorial on X10 and its implementation.
&nbsp; - X10 future plans.

Bio: David Grove currently co-leads the X10 project at IBM Research.
His primary research interests include the analysis and optimization
of object-oriented languages, virtual machine design and implementation,
JIT compilation, online feedback-directed optimization, and garbage collection.
He played a leading role in IBM's Jalapeno project and has been a member
of the Jikes RVM steering committee and core team since Jalapeno went
open source in 2001.&nbsp; From 2004 to 2008 he was a member of the Metronome
project and made significant contributions to the implementation of the
Metronome GC in IBM's WebSphere Real Time Java product.&nbsp; He has been
a Research Staff Member at IBM Research since 1998.&nbsp; He received his Ph.D.
from the University of Washington in 1998 and a B.S from Yale in 1992

Speaker:&nbsp;&nbsp; David Cunningham (IBM Research)
&nbsp;
&nbsp;&nbsp; Title:&nbsp;&nbsp; M3R: Increased Performance for In-Memory Hadoop Jobs

Abstract:

&nbsp; M3R is a new X10-based implementation of the Hadoop Map
&nbsp; Reduce (HMR) API targeted at online analytics on high
&nbsp; mean-time-to-failure clusters. It does not support resilience, and
&nbsp; supports only those workloads which can fit into cluster memory.
&nbsp; In return, it can run HMR jobs unchanged -- including jobs produced by
&nbsp; compilers for higher-level languages such as Pig, Jaql, and SystemML
&nbsp; and interactive front-ends like IBM BigSheets -- while providing
&nbsp; significantly better performance than the Hadoop engine on several
&nbsp; workloads (e.g. 45x on some input sizes for sparse matrix vector
&nbsp; multiply). M3R also supports extensions to the HMR API which can
&nbsp; enable Map Reduce jobs to run faster on the M3R engine, while not
&nbsp; affecting their performance under the Hadoop engine.

Bio:

David Cunningham's PhD work involved novel lock inference techniques and
correctness proofs for atomic sections in object-oriented languages
(Imperial College London, 2010).&nbsp; He has been employed by IBM for 3 years
as a software engineer working with the X10 team, responsible for X10RT
(the communications runtime) and CUDA backend / runtime implementation.&nbsp;
He has also participated
in general C++ backend and runtime implementation, X10 application
development, and X10 language design discussions.
